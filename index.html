模板模式

模版模式就是定一个操作中的骨架(步骤)，而将一些步骤延迟到子类中，template method可以使得子类不改变一些算法的结构，重新实现该算法的一些具体的实现，比如我们要加工一些饮品，如茶，咖啡，他们的步骤基本上就是加水，煮水，加材料，就可以制成饮品，他们的步骤基本上是一样的，可以把步骤抽象出来，然后实现去交给子类实现，这个就是模版模式。

###定义整体步骤
public abstract class AbstractMakeDrinks{
final void makeDrinks(){
addWater();
boilWater();
addMaterial();
}
abstract void addWater();
abstract void boilWater();
abstract void addMaterial();

}

###制作茶
public class MakeCoffee extends AbstractMakeDrinks {
@Override
void addWater() {
System.out.println(“添加水”);
}

@Override
void boilWater() {
    System.out.println("烧开");
}

@Override
void addMaterial() {
    System.out.println("添加咖啡");
}

}

###制作咖啡

public class MakeTea extends AbstractMakeDrinks{
@Override
void addWater() {
    System.out.println("添加牛奶");
}

@Override
void boilWater() {
    System.out.println("烧开");
}

@Override
void addMaterial() {
    System.out.println("添加茶叶");
}

}

###测试
public static void main(String[] args) {
MakeTea makeTea = new MakeTea();
makeTea.makeDrinks();
}

###输出结果

这样我们就可以不用在制作茶和咖啡的时候，都去实现makeDrinks方法，而是把它定义才抽象类中，实现代码的复用，当然，我们可以把烧开水的操作定义在抽象类中。如果我们需要加额外的东西，可以在抽象类中写一个钩子函数，类似于下面

public abstract class AbstractMakeDrinks {
final void makeDrinks(){
    addWater();
    boilWater();
    addMaterial();
    if(addMore()){ //是否需要添加额外的东西
        addition(); //由子类去实现
    }
}
abstract void addWater();
abstract void boilWater();
abstract void addMaterial();
abstract void addition();
public boolean addMore(){  
//默认为false，不添加，子类可以根据需要重写该操作
    return false;
    }
}

子类重新重写这些方法，就可以添加额外自己想要添加的东西。该算法的模版如果不希望子类去修改的话可以声明为final。

###优点
1.一次性实现一个的算法不变的部分，将可变的部分交给子类去实现
2.将子类公共的部分提取到父类中避免代码重复

###缺点
1.如果子类过多的话，会导致系统庞大
